What is PWA?

Progressive Web Application (PWA) is a type of web app that can operate both as a web page and mobile app on any device. It is a great solution for poor mobile UX and low conversion rates in your online store. Using standard technologies, PWA is aimed at delivering native-like user experience, with speedier conversion and cleaner browsing even with a poor Internet connection.

Types of Mobile Apps by Technology

There are three basic types of mobile apps if we categorize them by the technology used to code them:

  1. Native Apps: Applications specifically designed and developed for a particular mobile operating system, like IOS (using Swift/Objective-C) or Android (using Java/Kotlin).
    2. Pros:
    • Faster and more reliable in terms of performance.
    • Access to full device hardware and OS capabilities.
    • Offline capabilities and rich user experience.
    Cons:
    • Higher development cost as separate codebases are needed for different platforms.
    • Frequent updates and app store approval cycles.
    • Takes up space in the device.



  2. Hybrid Apps: Applications built using web technologies (HTML, CSS, JavaScript) wrapped inside a native container that allows them to run on different platforms with a single codebase (e.g., Cordova, Ionic).
    3. Pros:
    • Cross-platform, so they work on multiple platforms with a single codebase.
    • Quicker/cheaper to build.
    • Load quickly and less code to maintain.
    Cons:
    • Performance may not be good as native apps.
    • Limited access to device features compared to native apps.



  3. Progressive Web Apps (PWA): Web applications that behave like native apps on mobile devices. They are developed using web technologies (HTML, CSS, JavaScript) and provide enhanced functionalities such as offline access, push notifications, and device hardware integration.
    4. Pros:
    • Cross-platform compatibility with a single codebase.
    • No need for app store approvals and updates; updates happen in the background.
    • Cost-effective development.
    Cons:
    • Limited access to certain native devices features.
    • Browser-specific behaviour; may not be fully supported on older browsers.


Why PWA is the most suitable setup for modern standalone application

  • Development Cost: PWAs are highly cost-effective since they are developed once and can run on multiple platforms, unlike native apps that require separate codebases for iOS and Android.
  • Performance: While native apps offer the best performance, PWAs offer good enough performance for most use cases with modern browser APIs and are improving steadily.
  • Offline Functionality: Through service workers, PWAs can cache resources and data to work offline or under poor network conditions, which is a key requirement for modern apps.
  • User Experience: PWAs offer a near-native user experience with features like offline functionality, push notifications, and the ability to be installed on the home screen.
  • Deployment and Updates: PWAs do not require app store approvals, making deployment and updates seamless, which reduces maintenance overhead compared to native apps.

Exploring Progressive Web App

PWA Frameworks

  • React.js: Its component-based architecture, along with the support of tools like Create React App and Next.js, empowers developers to create high-performing PWAs.
  • Vue.js: Known for its simplicity and flexibility, Vue's ecosystem offers tools like Vue CLI, Vue Router, and Vuex, which streamline PWA development.
  • Angular: AngularJS offers powerful tools, progressive enhancements, TypeScript support, and a library of UI components to help you seamlessly develop a complex web application.
  • Svelte: Its simplicity and ability to generate lightweight, fast-loading PWAs have made it a rising star in the PWA development landscape.

Requirements of Files

A Secure Connection (HTTPS)

For the PWA purposes you need to provide a secure server with a HTTPS connection. This is the way to both protect user data and to build an extra layer of security within your website.

Service Workers

This is one of the key elements of PWA technology, which helps to determine how to handle network requests. A service worker is a client-side JavaScript file added to your codebase, which is working in the background to mimic app-like features.

A JSON file that contains metadata about the app (name, icons, theme color, etc.) and allows the app to be installed on the userâ€™s home screen.

Offline Cache Functional Application Example

A basic PWA that works offline has:

  • Service Worker: Registers and caches app shell assets like HTML, CSS, and JavaScript files.
  • IndexedDB or LocalStorage: Stores user data or session data locally.
  • Workbox: A powerful Google library that simplifies service worker creation and caching strategies.

Why PWA is a good fit for an application for QR scanning

  • Cross-platform support: The application can be accessed from any device with a browser, eliminating the need to develop separate native apps.
  • Offline Functionality: The service worker can cache the resources required for QR code scanning, allowing the app to function offline.
  • Low-cost maintenance: Since PWA doesn't require native app store distribution and approval processes, it allows faster updates and maintenance with minimal costs.

Mid-Fidelity Design for Attendance QR scan app

Purpose of the application

  • Primary Use: Users scan a QR code to mark attendance, even in offline mode.
  • Offline Capability: The app should cache necessary data and use an offline QR code scanning library.

Features:

  1. Home Screen:
    • Scan QR Code Button
    • Instruction for first-time users.
    • Option to refresh or sync data when online.
  2. QR Code Scanner:
    • Integrates an offline QR decoding library (like zxing-js/library).
    • Once a QR code is scanned, it checks if the user is registered and marks the attendance.
    • If offline, the app stores the scan data and syncs it when back online.
  3. Attendance Confirmation:
    • Show a success message after a QR code is successfully scanned.
    • Store scan history locally and sync when the network is available.
  4. Settings:
    • Option to clear cached data.
    • Sync attendance manually.
